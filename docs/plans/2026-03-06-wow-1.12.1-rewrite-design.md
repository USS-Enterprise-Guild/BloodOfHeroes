# BloodOfHeroes WoW 1.12.1 Strict Rewrite Design

## Goal

Rewrite the addon to run strictly on World of Warcraft 1.12.1 (Lua 5.0), with:

- World map markers only on zone maps for Eastern Plaguelands (EPL) and Western Plaguelands (WPL)
- Minimap markers only for nearby nodes
- No modern API usage

## Scope

In scope:

- Replace modern map/timer APIs with 1.12-compatible APIs and event flow
- Rebuild world map rendering for `WorldMapDetailFrame`
- Add minimap nearby-node rendering with zoom-aware distance
- Keep `/blood toggle`
- Add `/blood range <yards>` override for v0

Out of scope:

- Multi-client compatibility
- Third-party map libraries
- Automated test harness

## Architecture

### Core module

Use a single module in `Blood.lua` with a `BoH` table containing:

- Static node data
- Runtime state (`enabled`, pin pools, update timers, range override)
- Zone resolver and renderer functions

### Zone context

Use 1.12 map functions:

- `SetMapToCurrentZone()`
- `GetMapInfo()`
- `GetPlayerMapPosition("player")`

Normalize map names to internal keys:

- `EasternPlaguelands`
- `WesternPlaguelands`
- `nil` for all other maps

### World map renderer

- Parent world-map pins to `WorldMapDetailFrame`
- Rebuild pins only when current world map is EPL/WPL zone map
- Clear pins otherwise

### Minimap renderer

- Use an `OnUpdate` throttle (target 0.2s)
- Resolve current player position in zone coordinates
- Filter nodes by nearby distance
- Parent minimap pin frames to `Minimap`
- Reuse frames via a simple pool to reduce churn

## Data Model

Replace numeric `zoneID` with zone-keyed coordinates:

```lua
BoH.data = {
  EasternPlaguelands = { {x=35.9, y=57.4}, ... },
  WesternPlaguelands = { {x=7.1, y=50.7}, ... },
}
```

## Nearby Range Behavior

Default nearby distance is derived from minimap zoom:

- Read `Minimap:GetZoom()`
- Convert zoom level to a yard radius using a lookup table

Optional v0 override:

- `/blood range <yards>` sets fixed radius
- `/blood range reset` returns to zoom-derived radius

## Event and Refresh Flow

On login and zone/map-relevant events:

1. Resolve zone context
2. Refresh world-map pins
3. Start/stop minimap updates based on eligibility (enabled + EPL/WPL)

On minimap update tick:

1. Resolve player position
2. If invalid position, skip tick
3. Recompute nearby nodes and update minimap pin visibility/position

On `/blood toggle`:

1. Flip enabled state
2. Refresh/clear both renderers immediately

## Error Handling and Safety

- If position returns `0,0` or nil, skip minimap tick
- If map is not EPL/WPL, clear all pins
- Preserve user world map browsing by restoring map context if temporary context switching is needed
- Reuse pin frames and throttle updates for predictable performance

## Verification Plan (Manual)

1. `/blood toggle` enables and disables both world map and minimap pins
2. World map shows pins on EPL zone map
3. World map shows pins on WPL zone map
4. World map shows no pins on continent/world/other zones
5. Minimap shows only nearby nodes in EPL/WPL
6. Minimap nearby visibility changes when minimap zoom changes
7. `/blood range <yards>` overrides default range
8. `/blood range reset` restores zoom-derived range
